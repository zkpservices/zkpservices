# dApp Guide

Using zkp.services is simple.

All available services are accessible through the Dashboard once you have set up your account.

Below is a step-by-step guide on how to use each service, which includes:

1. **Request Data**
   Securely request a data snapshot and leverage custom MFA if needed.
2. **Request Update**
   Securely request a data update from another user or a smart device and leverage custom MFA if needed.
3. **Cross-Chain Backups**
   Back up your data with Web2/Web3 providers, including CCIP or EVM-compatible chains.
4. **Onboard to New Chain**
   Onboard your connected data source onto a new CCIP or EVM-compatible chain.

## Request Data

Before you get started, here are all the values you will need to Request Data:

<table>
  <tr>
    <th>VALUE</th>
    <th>DESCRIPTION</th>
  </tr>
  <tr>
    <td><b>Request</b></td>
    <td>This is the data you want to request.</td>
  </tr>
  <tr>
    <td><b>Key</b></td>
    <td>A secure key to help decrypt data once you receive it.</td>
  </tr>
  <tr>
    <td><b>Time Limit</b></td>
    <td>Time in seconds before the recipient must respond to your data request.</td>
  </tr>
  <tr>
    <td><b>2FA Information (optional)</b></td>
    <td>If you want added security, you can make it so that the responder has to undergo a 2FA requirement before responding.</td>
  </tr>
  <tr>
    <td><b>Response fee</b></td>
    <td>A data request will cost 10 ZKP tokens by default. If you require more ZKP tokens from whom you request data, increase the number.</td>
  </tr>
</table>

### STEP 1: Initiate a data request.

Once you are ready to initiate a data request, navigate to the Dashboard. Here you will find the "Request Data" card. Click it to open a modal that will allow you to request data.

### STEP 2: Fill out your data request.

You will be presented with the form below. Below are brief descriptions covering each input you must complete before requesting data.

<table>
  <tr>
    <th>INPUT</th>
    <th>DESCRIPTION</th>
  </tr>
  <tr>
    <td><b>Address of Receiver</b></td>
    <td>The wallet address containing the requested data.</td>
  </tr>
  <tr>
    <td><b>One Time Key</b></td>
    <td>Once you have received the requested data, the one-time key will be used to access it. You can put in your own or generate a random one. Either way, make sure you save it somewhere.</td>
  </tr>
  <tr>
    <td><b>Time Limit of Request</b></td>
    <td>The time limit dictates how long your request will be active for the requestee to respond in seconds.</td>
  </tr>
  <tr>
    <td><b>Require 2FA</b></td>
    <td>If selected, the requestee must complete 2 Factor Authentication ("2FA") to respond to your request. An additional set of inputs will be included if you decide to attach 2FA to your request.</td>
  </tr>
  <tr>
    <td><b>Set Response Fee</b></td>
    <td>The response fee will be the number of ZKP tokens the requestee requires to answer your request. </td>
  </tr>
</table>


Once you have completed the form, clicking “Call Smart Contract” will initiate the data request.

### STEP 3: Confirming your data request was sent.

Once you have initiated the data request, you will see that request in your recent activity at the bottom of the dashboard.

Navigate to the "Outgoing" tab, and you can check the status of your data request here.

## Cross-Chain Backups

Backing up your data to another chain is easy.

### STEP 1: Initiate a cross-chain sync.

Navigate to the dashboard to find the "Cross-Chain Backups" card. Click it to open a modal that will allow you to request an update to data.

Confirming your data has been backed up on the designated chain is as easy as swapping to that chain on your connected wallet, navigating to the Dashboard, and clicking the "Add Data" card to check the backed-up parameter, which should now be showing in the drop-down.
